Al Django User Serializer

Source code and details avaialble on my github account: https://github.com/d1Ng0/alSerializer
This is the front-end for the command line which takes some sets of personal data (name, address, phone number) and serialize them/deserialize them in at least 2 formats (pickle and hd5), and display it in at least 2 different ways (no need to use a GUI Framework - text output/HTML or any other human readable format is fine). There is no need to support manual data entry - you could manually write a file in one of your chosen formats to give you your input test data.
 The tool is provided with unit-test and cmd line tool: 
Usage: alSerCmd.py [-h] [--testDataSize TESTDATASIZE] [--filePath FILEPATH]
--format {pk,hd5}
User Serializer. Optional arguments:
-h, --help show this help message and exit
--testDataSize TESTDATASIZE, -t TESTDATASIZE Test the serializer with random users. Specify the batch size, default 50
--filePath FILEPATH, -p FILEPATH Path to save the file (without extension). Default "data/data"
--format {pk,hd5}, -f {pk,hd5} pk: pickle file format or hd5
